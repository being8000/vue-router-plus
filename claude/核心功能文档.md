# Framework7 路由系统：`f7routers`（Vue 适配层）与 `core/modules/router`（核心引擎）

Framework7 的路由系统分两层：**框架无关的核心路由引擎**（`src/core/modules/router`）负责路由匹配、历史管理、生命周期调度；**各前端框架适配层**（Vue/React/Svelte）负责把"要渲染的组件"实际挂载到框架自己的响应式渲染体系里。`f7routers` 就是 Vue 适配层里连接这两者的**注册表（registry）**。

## 一、`f7routers`（`src/vue/shared/f7.js:10-14`）

```js
const f7routers = {
  views: [],   // 所有挂载的 <f7-view> 实例
  tabs: [],    // 所有挂载的 <f7-tab> 实例
  modals: null,// 唯一的 <f7-routable-modals> 实例（单例）
};
```

它是一个**模块级全局单例**，没有任何路由逻辑，纯粹是"谁在哪儿"的登记簿：

- `views`：数组，因为一个 App 里可以有多个 `<f7-view>`（多 tab 视图、split view 等），每项形如 `{ routerId, el, instance, pages, setPages }`。
- `tabs`：数组，因为一个页面可以有多个 `<f7-tab>`，每项形如 `{ el, setTabContent }`。
- `modals`：单例对象，因为通常整个 App 只有一处 `<f7-routable-modals>` 容器承载所有 popup/sheet/popover 等组件化 modal。

`cleanup()`（`f7.js:22-29`）在 SSR 场景下把这三个集合清空，避免不同请求间状态串扰。

## 二、核心功能模块：`components-router.js`

这个文件（`src/vue/shared/components-router.js`）是真正的"桥梁逻辑"，通过 `proto` 的方式注入到 Router 原型上（见下文关联部分），核心是三组"加载/卸载"配对方法：

| 方法 | 作用 |
|---|---|
| `pageComponentLoader` | 在 `f7routers.views` 里按 `el`/`routerId` 找到目标 `<f7-view>`，把新页面数据 push 进 `viewRouter.pages`，调用 `setPages()` 触发 Vue 重渲染，再监听 `viewRouterDidUpdate` 事件，等 DOM 真正更新后 diff 出新增的 `.page` 元素，`resolve(pageEl)` 交还给 core router |
| `removePage` | 从 `viewRouter.pages` 里移除对应页面数据（`keepAlive` 页面则跳过销毁只做 DOM 保留） |
| `tabComponentLoader` / `removeTabContent` | 同样逻辑作用于 `f7routers.tabs` |
| `modalComponentLoader` / `removeModal` | 同样逻辑作用于 `f7routers.modals`（单例） |
| `openIn` | 转发给 `routerOpenIn`，处理"以 popup/sheet 打开某个 url"这种场景 |

**关键设计**：Vue 是虚拟 DOM + 响应式驱动，不能像 core router 那样直接 `$(el).append(...)` 操作真实 DOM。所以这里用"push 数据 → 触发响应式更新 → 等 Vue 更新完成的事件 → 从真实 DOM 里取回新元素"这套异步握手协议，用自定义事件 `viewRouterDidUpdate` / `tabRouterDidUpdate` / `modalsRouterDidUpdate` 做同步点。

## 三、与之配合的 Vue 组件

- **`view.vue`**：`onMounted` 时把自己 `{ routerId, el, pages, setPages }` push 进 `f7routers.views`；`onUpdated` 时 `emit('viewRouterDidUpdate', routerData)`；`onBeforeUnmount` 时从数组里 splice 掉自己。
- **`tab.vue`**：同样的注册/更新通知/注销三段式，作用于 `f7routers.tabs`。
- **`routable-modals.vue`**：`onMounted` 时把自己整个赋给 `f7routers.modals`（单例，不是 push），`onBeforeUnmount` 时置回 `null`。

三者都遵循同一个模式：**挂载注册 → 更新时广播事件 → 卸载注销**，这正是 `f7routers` 存在的意义——让 core router 能"盲发"加载请求，而不用关心当前是哪个具体的 Vue 组件实例在处理。

## 四、`src/core/modules/router` 核心功能

这是完全框架无关的路由引擎，纯 DOM + Dom7 操作：

- **`router.js`**：模块入口，作为 Framework7 模块系统的注册项，`create()` 时区分创建 **App Router**（`app.router`，全局，负责 URL 计算等）还是 **View Router**（`view.router`，每个 `<f7-view>` 一个，负责该 view 内的页面栈）。
- **`router-class.js`**：`Router` 类主体。
  - `currentRoute`/`previousRoute` 用 `Object.defineProperty` 实现，`set` 时触发 `routeChange` 事件；
  - `flattenRoutes()` 把嵌套/`tabs`/`detailRoutes` 路由配置打平成一维数组便于匹配；
  - `findMatchingRoute()`/`parseRouteUrl()`/`generateUrl()` 基于 `path-to-regexp` 做 URL ↔ Route 对象互转和匹配；
  - `pageCallback()` 统一分发 `page:init/beforeIn/afterIn/beforeOut/beforeRemove` 等页面生命周期事件，处理 `keepAlive`、滚动位置恢复等；
  - `saveHistory`/`restoreHistory`/`init()` 处理浏览器历史（`browserHistory`）与内部 `history` 数组的同步；
  - `removePage`/`removeTabContent`/`removeModal`/`removeEl` 是**默认**的、基于纯 DOM 操作的移除实现。
- **`navigate.js`**：`navigate()` 是导航入口——解析目标、匹配路由、按 `component`/`url`/`content`/`el` 等不同加载方式分派，其中当路由内容是"组件"时会调用 `router.pageComponentLoader(...)`（`navigate.js:593`）。
- **`tab.js`**：`tabLoad`/`tabRemove`，加载 tab 路由内容时调用 `router.tabComponentLoader(...)`。
- **`modal.js`**：`modalLoad`/`modalRemove`，加载 popup/sheet/popover/panel 等路由时调用 `router.modalComponentLoader(...)`。
- **`component-loader.js`**：`pageComponentLoader`/`tabComponentLoader`/`modalComponentLoader`/`componentLoader`/`openIn` 的**默认实现**——针对纯 F7 Component（字符串模板 + XHR 加载）场景。
- 其余：`back.js`（后退）、`swipe-back.js`（iOS/MD 手势返回）、`clear-previous-history.js`、`redirect.js`、`process-route-queue.js`（导航队列串行化防止并发冲突）、`async-component.js`、`app-router-check.js`（限定某些 API 只能在 App Router 上调用）。

## 五、两者的关联——"协议方法"插件机制

关键在于这几个方法名是**约定的钩子接口**：`pageComponentLoader`、`tabComponentLoader`、`modalComponentLoader`、`removePage`、`removeTabContent`、`removeModal`、`openIn`。

- Core router（`navigate.js`/`tab.js`/`modal.js`）只负责**决定何时调用**这些方法，默认实现在 `component-loader.js` 里（针对无框架/字符串模板场景）。
- Vue 插件安装时（`src/vue/shared/plugin.js:30`）执行：
  ```js
  Framework7.Router.use(componentsRouter);
  ```
  把 `components-router.js` 里的 `proto` 方法**覆盖**到 `Router` 原型上，替换掉默认实现。
- 被覆盖后的实现内部依赖 `f7routers` 去反查"当前这个 core Router 实例对应的是哪个 Vue `<f7-view>`/`<f7-tab>`/`<f7-routable-modals>`"，然后调用其 `setPages`/`setTabContent`/`setModals` 驱动 Vue 响应式更新。

**一句话总结**：`src/core/modules/router` 是框架无关的路由状态机和生命周期调度中心；`f7routers` 是 Vue 适配层专属的"实例注册表"，通过 `Framework7.Router.use()` 注入的钩子方法，把 core router 的加载/卸载请求路由到具体的 Vue 组件实例上，是纯 JS 路由引擎与 Vue 响应式渲染系统之间的桥梁。React、Svelte 各自也有一套同构的 `plugin.js` + 组件注册表机制（`src/react/shared`、`src/svelte/shared`），设计模式完全一致。
